# Nielsen's 10 Usability Heuristics Checklist

## H1: Visibility of System Status

- [ ] The system provides immediate feedback for every user action (clicks, form submissions, state changes)
- [ ] Progress indicators are shown for operations that take longer than 1 second
- [ ] The current state of the system is always visible (active page, selected item, connection status)
- [ ] Upload and download operations show progress percentage and estimated time remaining

## H2: Match Between System and the Real World

- [ ] Labels and terminology use the user's language, not developer jargon or internal codenames
- [ ] Icons and metaphors are familiar and culturally appropriate for the target audience
- [ ] Information is presented in a natural, logical order that matches real-world conventions

## H3: User Control and Freedom

- [ ] Undo and redo are supported for destructive or significant actions
- [ ] Users can cancel or exit any process without losing unrelated work
- [ ] Confirmation dialogs protect against irreversible actions (delete, send, publish)
- [ ] Navigation allows users to go back without losing form data

## H4: Consistency and Standards

- [ ] UI elements (buttons, links, icons) behave the same way throughout the application
- [ ] Terminology is consistent -- the same concept is never referred to by different names
- [ ] Platform conventions are followed (e.g., standard keyboard shortcuts, native UI patterns)
- [ ] Visual styling (colors, spacing, typography) is uniform across all screens

## H5: Error Prevention

- [ ] Destructive actions require a confirmation step or undo period
- [ ] Form inputs use constraints (date pickers, dropdowns, input masks) to prevent invalid data
- [ ] Inline validation warns users before they submit forms with errors

## H6: Recognition Rather Than Recall

- [ ] Recently used items, searches, and actions are easily accessible
- [ ] Help text and labels are visible in context -- users do not need to memorize instructions
- [ ] Options are presented for selection rather than requiring free-text input where possible

## H7: Flexibility and Efficiency of Use

- [ ] Keyboard shortcuts are available for frequent actions
- [ ] Power users can customize or skip steps in workflows
- [ ] Default values are provided for common scenarios to reduce input effort
- [ ] Batch operations are available for repetitive tasks

## H8: Aesthetic and Minimalist Design

- [ ] Each screen shows only information relevant to the current task
- [ ] Visual hierarchy clearly distinguishes primary actions from secondary ones
- [ ] Decorative elements do not compete with functional content

## H9: Help Users Recognize, Diagnose, and Recover from Errors

- [ ] Error messages are written in plain language and explain what went wrong
- [ ] Error messages suggest a concrete corrective action
- [ ] Errors are displayed near the relevant input field, not only at the top of the page
- [ ] The system preserves user input when an error occurs so they do not have to re-enter data

## H10: Help and Documentation

- [ ] Contextual help is available where users are likely to need it (tooltips, inline hints)
- [ ] Documentation is searchable and task-oriented
- [ ] Onboarding flows guide new users through core features without overwhelming them
